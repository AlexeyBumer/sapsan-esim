# Этап 1 — Регистрация и вход (Supabase)

Готово в коде: страницы регистрации `/register`, входа `/login`, личный кабинет `/account` (защищён), подтверждение email, выход. Баланс и рефералы пока показаны как заглушки — это следующие этапы.

Чтобы это заработало, нужно настроить Supabase и добавить ключи в Vercel.

## 1. Настройка Supabase

1. Создайте проект на `supabase.com` (если ещё нет). Регион — Frankfurt.
2. **Выполните SQL:** откройте файл `supabase-setup.sql` из проекта, скопируйте всё содержимое, вставьте в Supabase → **SQL Editor** → нажмите **Run**. Это создаёт таблицу профилей, правила безопасности (RLS) и авто-создание профиля при регистрации.
3. **Email-подтверждение:** Authentication → Providers → Email → убедитесь, что включены Email и «Confirm email».
4. **URL:** Authentication → URL Configuration:
   - Site URL: `https://www.sapsansim.com`
   - Redirect URLs: добавьте `https://www.sapsansim.com/**`

## 2. Ключи в Vercel

Settings → Environment Variables. Возьмите значения из Supabase → Settings → API:

| Key | Value | Видимость |
|-----|-------|-----------|
| `NEXT_PUBLIC_SUPABASE_URL` | Project URL (`https://xxxxx.supabase.co`) | публичный |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | ключ `anon public` | публичный |
| `SUPABASE_SERVICE_ROLE_KEY` | ключ `service_role` | **СЕКРЕТНЫЙ** |

⚠️ `service_role` — самый важный секрет. Он даёт полный доступ к базе в обход всех правил. Только в Vercel, никогда в код, никогда в чат, никому.

Для `NEXT_PUBLIC_`-переменных Vercel может показать «Mark as Safe» — это нормально, жмите (они и так публичны).

После добавления — **Redeploy**.

## 3. Проверка

1. Откройте `https://www.sapsansim.com/register`, зарегистрируйтесь.
2. Должна появиться надпись «Проверьте почту». Придёт письмо со ссылкой — перейдите по ней.
3. После подтверждения войдите на `/login`. Должно перебросить в `/account`.
4. В кабинете виден ваш email, нулевой баланс и реферальный код.
5. Проверьте защиту: выйдите и попробуйте открыть `/account` напрямую — должно перекинуть на `/login`.

## Безопасность — что уже сделано

- Пароли хранит Supabase в хешированном виде. Мы их не видим и не храним.
- **RLS включён**: каждый пользователь читает только свой профиль, чужие данные недоступны даже при попытке запроса.
- Баланс и реферальный код **пользователь менять не может** — это поля только для сервера (service_role). На следующих этапах их будет менять только серверный код после проверки реальной оплаты.
- Личный кабинет защищён middleware: без входа туда не попасть.
- Подтверждение email включено — нельзя зарегистрироваться на чужую почту и пользоваться без доступа к ней.

## Что вы должны сделать для безопасности (важно)

- Включите двухфакторную аутентификацию (2FA) на аккаунтах: Supabase, Vercel, GitHub, ваша почта. Это защищает от угона самих сервисов.
- Никому не передавайте `service_role`-ключ.
- Пароль от базы данных Supabase храните в надёжном месте (менеджер паролей).
