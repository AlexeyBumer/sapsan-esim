# Подключение оплаты (Heleket) — инструкция

Платёжная часть уже написана в коде. Чтобы она заработала на живом сайте, нужно вставить два секретных ключа Heleket в настройки Vercel. **Ключи нигде в коде не хранятся** — только в защищённом хранилище Vercel.

## Как работает (поток)

1. Клиент на странице `/order` выбирает: новая eSIM или пополнение, вводит email (или ID eSIM), выбирает пакет (5–50 ГБ) или своё значение.
2. Жмёт «Перейти к оплате» → сайт создаёт счёт в Heleket → клиента перебрасывает на страницу оплаты Heleket.
3. После успешной оплаты Heleket возвращает клиента на `/success` с его **ID заказа** (он же ID eSIM).
4. На странице успеха клиент видит свой ID и кнопку «Написать оператору @sapsansup» (с заранее заполненным текстом). Отправляет скрин оплаты + ID.
5. Оператор вручную активирует/пополняет eSIM и записывает ID.

## Что нужно сделать вам (один раз)

### 1. Получить ключи Heleket
Зайдите в `dash.heleket.com` → раздел **API** → создайте/скопируйте:
- **Merchant ID**
- **Payment API key**

### 2. Вставить ключи в Vercel
1. Откройте проект на `vercel.com` → вкладка **Settings** → раздел **Environment Variables**.
2. Добавьте три переменные (кнопка Add):

| Name | Value |
|------|-------|
| `HELEKET_MERCHANT_ID` | ваш Merchant ID |
| `HELEKET_PAYMENT_API_KEY` | ваш Payment API key |
| `NEXT_PUBLIC_SITE_URL` | `https://www.sapsansim.com` |

3. Сохраните. Зайдите во вкладку **Deployments** → у последнего деплоя нажмите **Redeploy**, чтобы переменные применились.

### 3. Настроить URL возврата в Heleket (если потребует)
В большинстве случаев URL возврата и вебхука уже передаются из кода автоматически:
- Возврат после оплаты: `https://www.sapsansim.com/success`
- Вебхук о статусе: `https://www.sapsansim.com/api/heleket-webhook`

Если в кабинете Heleket есть отдельные поля для этих адресов — впишите их туда же.

## Проверка

1. Откройте `https://www.sapsansim.com/order`.
2. Выберите пакет, введите тестовый email, нажмите «Перейти к оплате».
3. Должна открыться страница оплаты Heleket. Если открылась — связка работает.
4. Если появилась ошибка «Не заданы HELEKET_…» — значит, переменные не добавлены или не сделан Redeploy.

## Важно про безопасность

- Никогда не вставляйте `HELEKET_PAYMENT_API_KEY` в код, в публичные файлы или в чаты. Только в Environment Variables Vercel.
- Файл `.env` добавлен в `.gitignore` — он не попадёт на GitHub.
- Создание счёта и проверка оплаты происходят на сервере (в `/api/checkout` и `/api/heleket-webhook`), ключ наружу не передаётся.
