# Миграции базы данных Sapsan eSIM

Эти файлы — снимок того, что реально выполнено в Supabase → SQL Editor
по состоянию на 22 июня 2026. Они добавлены в git **задним числом**,
для истории и на случай восстановления базы с нуля — сами таблицы
в продакшен-Supabase уже существуют, повторно их запускать не обязательно.

Все скрипты написаны идемпотентно (`if not exists`, `drop policy if exists`),
поэтому повторный запуск безопасен и ничего не сломает.

## Порядок (если разворачиваете базу с нуля)

1. `001_profiles_init.sql` — таблица `profiles`, генерация реферального кода,
   автосоздание профиля при регистрации (Этап 1).
2. `002_orders_admin_sessions.sql` — таблицы `orders` и `admin_sessions` (Этап 2).
3. `003_fix_profiles_balance_rls.sql` — фикс уязвимости из аудита 21 июня:
   убирает возможность пользователя менять себе баланс/реф-код напрямую.
4. `004_orders_referral_code.sql` — добавляет колонку `referral_code` в `orders`.

## На будущее

Любую новую SQL-правку, которую делаете прямо в Supabase SQL Editor,
сохраняйте сюда же отдельным пронумерованным файлом (`005_...sql` и т.д.)
сразу после применения — так схема БД больше никогда не "потеряется",
в отличие от того, что произошло в этот раз.
